> *30 years ago: computers were boxes running Netscape on 56k modems. 30 years from today: every device is a PLATO room. The same concept, different scale.*

---

## 1996 — The Tip of THAT Iceberg

In 1996:
- The web was static pages. JavaScript was born that year.
- MUDs ran on university mainframes over dial-up.
- The original PLATO system (1960s) was already 30 years old — plasma displays, touch screens, online communities, bitmap graphics. All before their time.
- Nobody saw where it was going.

What was actually forming:
- Static pages → dynamic web → web apps → cloud computing
- MUDs → graphical MMOs → virtual worlds → metaverse
- Mainframe PLATO → personal computer → smartphone → ambient computing

**The tip of the 1996 iceberg looked like chat rooms and static HTML. The mass below was the entire internet economy.**

## The Three Eras of PLATO

### PLATO v1 (1960s-1980s) — Mainframe Education
*Terminals. Plasma displays. Bitmap graphics. Touch screens. Online communities.*

Nobody in 1965 called it "the future of computing." They called it "that teaching terminal at UIUC." But it had:
- Multi-user real-time interaction (preceded the internet)
- Vector graphics (preceded GUIs)
- Touch input (preceded the iPhone by 40 years)
- Online forums, chat, multiplayer games (preceded everything)

The tip of THAT iceberg looked like a teaching tool. The mass below was the blueprint for interactive computing.

### PLATO v2 (2020s-2030s) — The Fleet
*Rooms. Tiles. Embeddings. Calibration. The field.*

Nobody in 2026 calls it "the future of computing." They call it "that MUD with the steampunk crabs" or "FM's constraint theory thing." But it has:
- Universal room format (any device, any language, any transport)
- Tile = embedding = constraint check = radio packet (one format, all purposes)
- Calibration core (coordination through shared timing, not messages)
- The continuous field (knowledge as space, not storage)

The tip looks like a collection of repos. The mass below is the computing model of 2056.

### PLATO v2056 (2030s-2050s) — The Ambient Field
*Knowledge as the substrate. Intelligence as a property of the field. Rooms as the universal interface.*

In 2056:
- You don't "open an app." You walk into a room.
- You don't "save a file." The tiles persist in the field.
- You don't "connect to a network." FLUX adapts automatically between any two nodes.
- You don't "run an AI model." Intelligence is a property of the field, not a separate entity.
- You don't "install an update." The room evolves as tiles accumulate.
- You don't "back up data." The holographic field means every tile contains information about the whole.

## The Invariant

The original PLATO (1960s) had:
```
64 terminals × plasma displays = one room, 64 learners
```

Our PLATO (2020s) has:
```
64 bytes × floating embeddings = one tile, 8 dimensions
```

The 2056 PLATO will have:
```
64 dimensions × ambient field = one universe, infinite rooms
```

**64 is the invariant.**
- 1965: 64 terminals in the original PLATO classroom
- 2026: 64-byte tiles = 1 cache line = 1 zmm register = 1 constraint check
- 2056: 64-dimensional embedding space as the universal substrate

The number didn't change. The scale did. 64 terminals in a room became 64 bytes in a tile became 64 dimensions in the field. The PLATO concept — rooms that think, tiles that remember, agents that walk between them — is the same at every scale for the last 60 years. The next 30 will make it invisible, like the web is today.
